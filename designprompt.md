РОЛЬ
Ты — senior frontend-разработчик и продуктовый дизайнер с опытом работы в линейных, чистых SaaS-интерфейсах (стиль Linear, Vercel, Notion, Attio, Pipedrive). Твоя задача — спроектировать и сверстать production-ready дашборд для B2B-сервиса, генерирующего лиды для агентства коммерческой недвижимости.
ЗАДАЧА
Собери одностраничное веб-приложение (SPA) — дашборд /leads/ для сервиса Lead Intelligence. Сервис ищет компании, через AI определяет их потребность в коммерческой недвижимости (офис, склад, торговая точка, производство), ставит каждой компании score 0–100 с объяснением и складывает в очередь, с которой работает менеджер.
СТЕК

React 18 + TypeScript (функциональные компоненты, хуки)
Tailwind CSS для стилизации (без UI-китов вроде MUI/Antd — только Tailwind + shadcn/ui компоненты при необходимости)
lucide-react для иконок
recharts для графиков
Один файл App.tsx со всем кодом — чтобы можно было сразу запустить
Mock-данные внутри компонента (массив из 8–12 лидов)
Никаких внешних API-вызовов, никаких бэкенд-запросов — только UI + локальный state

ВИЗУАЛЬНЫЙ ЯЗЫК
Стиль: flat, чистый, продуктовый. Это не лендинг и не презентация — это рабочий инструмент, в котором менеджер сидит 8 часов в день.
Запрещено:

Градиенты, тени (кроме hover/focus), неоновые эффекты, glow
Скруглённые углы больше 12px
Иконки emoji вместо настоящих иконок
Капс, заголовки в Title Case
Шрифты тяжелее 500 (никаких bold 700)
Декоративные иллюстрации, фоновые паттерны

Обязательно:

Светлая тема + поддержка dark mode через Tailwind dark:
Шрифт: системный sans-serif (Inter или -apple-system)
Размер заголовков: h1=22px, h2=18px, h3=16px, body=14px, caption=12px
Все веса — 400 или 500, никаких 600/700
Border-radius: 8px для элементов, 12px для карточек
Бордеры: 1px solid с очень низкой непрозрачностью (типа border-zinc-200 dark:border-zinc-800)
Отступы — щедрые, воздух между блоками
Sentence case везде, в том числе в кнопках и заголовках

ПАЛИТРА
Используй нейтральную базу (zinc/neutral) + 2–3 акцента:

Нейтральный: zinc-50/100/200/500/700/900
Акцент основной: blue-600 (для primary действий)
Семантика: emerald-600 (success/score 80+), amber-600 (warning/score 60–79), rose-600 (danger/мусор)

Score-цвета строго по диапазонам:

80–100 → emerald
60–79 → amber
<60 → zinc (приглушённый)

Левая часть (avatar + content):

ДОСТУПНОСТЬ

Все интерактивные элементы — настоящие <button> с фокус-кольцом
Иконки декоративные — aria-hidden
Иконки-кнопки — aria-label
Контраст текста на цветных бейджах — минимум AA (тёмный текст на светлой плашке того же цветового семейства)

КАЧЕСТВО КОДА

Один файл App.tsx, экспортирует default function App()
Mock-данные вынеси в массив const LEADS: Lead[] = [...] сверху файла, тип Lead опиши явно
Сценарии — тоже в массив const SCENARIOS
Логика фильтрации через useMemo
Состояние — useState, не тяни Redux/Zustand
Никаких inline-стилей, кроме случаев, когда Tailwind физически не справляется
Комментарии на русском в ключевых местах
